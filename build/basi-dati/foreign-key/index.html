Vincoli di integrità referenziale

  1. Principi generali

Nell'ambito delle basi di dati relazionali, possiamo definire particolari vincoli di integrità referenziale tra le diverse tabelle che compongono la nostra base dati. Più nello specifico, è possibile esplicitare delle relazioni tra le varie entità della base dati. Vediamo quindi un esempio.

Tabella auto (A)

IDMARCA
1AUDI
2CHEVROLET
3BMW

Tabella modello (M)

IDMARCA
1CAMARO SS
2CRUZE
3CAPTIVA

IDMARCA
1BENZINA
2DIESEL
3ELETTRICA

Auto configurata (AC)

IDMarca_IdModello_IdCarburante_Id
1211

La tabella Auto configurata contiene le referenze alle tabelle precedenti, ossia l'Id specifico del record che identifica:

  • la marca dell'automobile;
  • il modello dell'automobile;
  • il tipo di carburante.

Nel nostro caso specifico si tratta di una CHEVROLET CAMARO SS BENZINA. Diremo quindi che gli indici della tabella Auto configurata sono delle Foreign-Key (chiavi esterne) delle rispettive tabelle che precedono. Ora però la tabella Auto configurata contiene solamente le referenze alle altre tabelle, e non, i rispettivi valori. Volendo recupare i valori dobbiamo utilizzare delle istruzioni di JOIN.

  1. Codice SQL
SELECT AC.ID, 
       A.MARCA AS MARCA_AUTO, 
       M.MARCA AS MODELLO_AUTO, 
       C.MARCA AS CARBURANTE_AUTO
FROM "Auto configurata" AC
JOIN "Tabella auto" A ON AC.Marca_Id = A.ID
JOIN "Tabella modello" M ON AC.Modello_Id = M.ID
JOIN "Tabella carburante" C ON AC.Carburante_Id = C.ID;


Il risultato di questa query sarà una tabella che mostra tutti i valori (proprietà) dell'auto configurata.
